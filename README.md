# 📚 Padrões de Projeto (Design Patterns)

## 🔍 O que é?
**Padrões de projeto** são soluções reutilizáveis para problemas frequentes no desenvolvimento de software. Funcionam como templates adaptáveis, não como código pronto para somente copiar e usar.

Eles são subdivididos em 3 grupos **Criacionais**, **Estruturais** e **Comportamentais**.

| Tipo | Descrição |
|---------|-----------|
| **Criacionais** | Fornecem vários mecanismos de criação de objetos, que aumentam a flexibilidade e reutilização de código já existente |
| **Estruturais** | Explicam como montar objetos e classes em estruturas maiores mas ainda mantendo essas estruturas flexíveis e eficientes |
| **Comportamentais** | Voltados aos algoritmos e a designação de responsabilidades entre objetos |

### 📌 Principais Características
| Aspecto | Descrição |
|---------|-----------|
| **Propósito** | Descreve o problema e a possível solução |
| **Motivação** | Explica a fundo o problema e a solução |
| **Estrutura** | Mostram como cada parte de uma classe funcionca e se comunica com outra |

## 📚 Objetivo
O principal propósito desse repositório é crescer com um tempo e conter uma breve explicação do padrão, para que serve, qual problema soluciona e por que soluciona.
Este repositório tem o intuido de ser um guia de consulta de todos os padrõe de projetos mais utilizados no mundo do desenvolvimento e arquitetura de Software.
